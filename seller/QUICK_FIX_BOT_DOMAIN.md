# Быстрое решение: "Bot domain invalid"

## Проблема
Ошибка "Bot domain invalid" означает, что домен вашего приложения не привязан к боту в BotFather.

## Решение за 3 шага

### Шаг 1: Откройте BotFather
1. Откройте Telegram
2. Найдите [@BotFather](https://t.me/BotFather)
3. Отправьте команду `/setdomain`

### Шаг 2: Выберите бота и домен
1. Выберите вашего бота из списка
2. Введите домен:
   - **Для локальной разработки:** `localhost`
   - **Для продакшена:** ваш домен (например: `seller.vibelook.com`)
3. **Важно:** вводите только домен, без `http://`, `https://` или порта

### Шаг 3: Перезапустите приложение
```bash
# Остановите приложение (Ctrl+C)
# Запустите снова
npm run dev
```

## Проверка

1. Откройте страницу авторизации: `http://localhost:3001/auth`
2. В режиме разработки вы увидите диагностическую информацию с текущим доменом
3. Убедитесь, что домен в BotFather совпадает с доменом в диагностике

## Примеры правильных доменов

✅ **Правильно:**
- `localhost`
- `seller.vibelook.com`
- `vibelook.com`

❌ **Неправильно:**
- `http://localhost`
- `https://localhost`
- `localhost:3001`
- `http://seller.vibelook.com`

## Если проблема не решена

1. Проверьте консоль браузера (F12) на наличие ошибок
2. Убедитесь, что `NEXT_PUBLIC_TELEGRAM_WIDGET_BOT_USERNAME` в `.env.local` совпадает с username бота (без @)
3. Очистите кэш браузера (Ctrl+Shift+Delete)
4. Проверьте, что токен бота правильный в `.env.local`
